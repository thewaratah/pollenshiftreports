# analytics-viewer.html

**Location:** `THE WARATAH/SHIFT REPORT SCRIPTS/analytics-viewer.html`
**Type:** Bundled React sidebar application
**Opened via:** Menu > Analytics > Open Analytics Viewer (password-protected)

---

## What This File Does

This is a pre-built React application bundled into a single HTML file. It provides an interactive analytics viewer that opens as a sidebar inside the Google Sheets spreadsheet.

The file contains minified React, Tailwind CSS, and Lucide icons all inlined into one self-contained HTML page — no external dependencies are loaded at runtime. This is the standard approach for shipping a React UI inside Google Apps Script, since GAS doesn't support npm packages or external build pipelines at runtime.

---

## How It's Used

1. A user opens the Waratah shift report spreadsheet
2. They go to the custom menu and select **Open Analytics Viewer** (requires admin password)
3. `UIServer.js` calls `HtmlService.createHtmlOutputFromFile('analytics-viewer')` and opens it as a sidebar
4. The React app renders inside the sidebar panel

## Important Notes

- **Do not edit this file by hand.** It's a build artifact. The readable source code lives in a separate React project that gets compiled and bundled into this single HTML file.
- **No `google.script.run` calls detected** in the bundled output — the viewer may render analytics from data passed in at load time, or it may be a static dashboard view.
- The file is password-gated through `pw_openAnalyticsViewer()` in `Menu.js`, so only admin users can access it.
